参考教程：Create Let’s Encrypt Wildcard Certificates in NGINX

Let’s Encrypt 是免费、开放和自动化的证书颁发机构由 Linux基金会(Linux Foundation)进行日常管理维护，它为 1.8 亿个网站提供TLS证书的非盈利性证书颁发机构, 通过它我们可以免费申请网站证书，并在网站上启用 HTTPS (SSL/TLS) 提供支持。

参数说明：

  • certonly: 这个子命令告诉 certbot 只获取证书，而不尝试安装它（例如，不会自动配置 web 服务器如 Apache 或 Nginx）。
  • --manual: 这个选项指示 certbot 使用手动模式，这意味着用户将需要手动执行某些步骤，如在 DNS 记录中添加验证记录。
  • --preferred-challenges=dns: 告诉 certbot 使用 DNS 来验证域名的所有权。这涉及到在域名的 DNS 记录中添加一个特定的 TXT 记录。
  • --server https://acme-v02.api.letsencrypt.org/directory: 指定了 ACME 服务器的 URL。ACME（自动证书管理环境）是 Let’s Encrypt 使用的协议。v02 是协议的版本。
  • --agree-tos: 这个选项表示用户同意服务条款。在自动化模式下运行时，此选项可以避免手动干预。
  • -d "*.wzhecnu.cn": -d 选项指定了要为其获取证书的域名。在这个例子中，它将获取一个通配符证书，这意味着证书将对 wzhecnu.cn 域名下的所有子域名有效（例如 subdomain.wzhecnu.cn）。

使用 --manual 选项时，certbot 将提供一些指示，告诉需要添加的 DNS 记录。一旦按照指示添加了 DNS 记录，并且它们已经生效（这可能需要一些时间），certbot 将能够验证用户对域名的控制并发放证书。
