Lets Encrypt 配置泛域名证书

Let’s Encrypt 是一个免费、开放且自动化的证书颁发机构，由 Linux 基金会进行日常管理维护。它为大量网站提供 TLS 证书，帮助网站轻松实现 HTTPS 加密。下面我将介绍如何利用 Let’s Encrypt 为网站生成免费的泛域名 SSL 证书。

安装 Certbot

首先，我们需要安装 Certbot。Certbot 是一个自动化的工具，用于获取和续签 Let’s Encrypt 的 SSL 证书。

获取泛域名证书

接下来，我们将通过 Certbot 获取泛域名证书。泛域名证书允许我们为一个域名及其所有子域名使用同一个 SSL 证书。

在这个过程中，Certbot 会指导你通过添加 DNS TXT 记录来验证你对域名的控制权。完成验证后，证书将被颁发，并保存在 /etc/letsencrypt/live/wzhecnu.cn/ 目录下。

参数释义：

  • certonly: 仅获取证书，不尝试安装。
  • --manual: 手动模式，需手动执行一些步骤，例如添加 DNS 记录。
  • --preferred-challenges=dns: 使用 DNS 验证域名所有权。
  • --server: 指定 ACME 服务器的 URL。
  • --agree-tos: 同意服务条款。
  • -d: 指定获取证书的域名。

上边是针对一般域名托管平台可用的方案（比如阿里云）。运行命令过程中，需添加对应的解析记录。
